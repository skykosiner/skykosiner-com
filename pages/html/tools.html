Tools I'm using

Software

  • Dwm

    For my window manger of choice I use Dwm. This allows me to edit the source code and get more and more options then I can get with i3 or anything else.

  • Dwmblocks

    By defualt in dwm you need to use xsetroot -name in a while loop to update and set your statusbar. I don't like this as i don't need everything in my statusbar updating every second, and with some things I want to send a kill signal to update it. Dwmblocks allows me to change the update time of each script, and with some scripts such as volume it only updates when a kill signal is sent.

  • St

    Along with using suckless for my window manger I also use suckless for my terminal with st. I like st as I can make it do anything I want compared to what I was using before Alacritty. The only issue is out of the box st is not very usebul, in fact you can't even scrollback.

  • Doom Emacs

    I come from the path of vs code, to vim/neovim, down to emacs. I've alwayws been very spetical of emacs over vim as it just seemed like more work for no reason. But after trying it out with org mode, org agenda, and just editing code. I found I really do like it. I do miss how easy it was to add anything to neovim with lua, as I still don't really know lisp. But once I put in the effort to learn lisp just like in neovim I'll be able to add anything I want.

  • Qutebrowser

    I used brave for almost two years and before that firefox. But even with 16gb of ram I always have too many tabs open and brave takes up too much ram. I found the qutebrowser a few weeks ago, and it's keyboard driven with vim keybinds, and barley eats any ram even with loads of tabs open.

  • Lf

    I prefear to do everything I can in a terminal or emacs. To do file mangement and quickly find files I use lf with a bunch of little scripts I have writen my self. The bueaty of lf is that you can use any shell script to make it do anything with a file which is really nice
